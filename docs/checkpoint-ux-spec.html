Cookrew · engineering spec · checkpoint UX v1

Checkpoints: timeline, context sync & dual titles

The checkpoint list is 1:1 with the agent's own history — now the terminal view gets a timeline scrollbar that makes every checkpoint a place you can jump to, act on, and read two ways.

source: checkpoint-ux-program-spec-u mapping: Forge component/sync: Velvet visuals: Fresco eval: Magpie

00 Principle & foundation (already shipped)

A checkpoint is one user-intent boundary: one real Enter, one prompt→reply exchange, one restore point. Capture-side integrity is done — Shift+Enter (ESC+CR) appends a newline instead of minting phantoms (15d48f1), boot phantoms are discarded, and the list is verified 1:1 against the agent's native /rewind history. This spec is the presentation program on top of that foundation: items 2–4 of the note.

02 Context follows checkpoint Forge Velvet

Selecting a checkpoint scrolls the black context view to that turn and highlights its span; scrolling the context steps the active checkpoint. Bi-directional, no feedback loops.

Forge mapping contract (already landing in shared/turn.ts)

FieldWhereMeaning
TurnRecord.scrollLineturns API0-based scrollback line where the checkpoint's prompt echo begins (tracker snapshot at turn start). Best-effort: absent when the tracker never saw the start.
activity.turnStartLineactivitiesScrollback line where the LIVE turn began; null outside a running turn. Same coordinate space.
  • Coordinate space: pane-width-wrapped lines as the PTY attach replay serves them. A resize re-wraps — treat offsets as best-effort and clamp; a screen clear invalidates older offsets (consumers clamp to buffer start).
  • Highlight span = [ckpt[i].scrollLine, (ckpt[i+1].scrollLine ?? turnStartLine ?? bufferEnd)).
  • Sync guard (Velvet): one owner at a time — a programmatic jump sets a syncing flag that suppresses the scroll listener until the scroll settles (or ~150 ms), and vice-versa. Never re-enter.
  • Scroll→checkpoint stepping: the active checkpoint is the last one whose scrollLine ≤ viewportTop + ⅓·viewportHeight — the reader's eye line, not the exact top.
  • Records with no scrollLine (pre-feature history, missed starts) are still listed and jumpable — jump falls back to prompt-echo text search in the buffer, else no-op with the stop still selectable for actions.

03 Dual title modes Velvet

Every checkpoint reads two ways — a user toggle, persisted:

  • CONCLUSION — the Sous title (what the turn accomplished). Fallback: first line of the prompt when no title exists.
  • PRECISE PROMPT — the exact user input, newlines preserved (the 1:1 fix records multiline prompts verbatim, so this mode is now honest). Render with white-space:pre-line, clamp to ~3 lines, full text on hover/title.
  • The toggle applies everywhere checkpoints are titled: timeline popovers, history panel rows, card pager meta. One preference, one key (e.g. checkpointTitleMode in the persisted UI prefs; phone follows via the same remote-api surface).

04 The timeline scrollbar Velvet Fresco

A vertical checkpoint timeline docked to the context view replaces the ◀/▶ arrow pager. Each stop is a checkpoint diamond (the existing badge language); the live head is a pulsing dot. Press a stop → the context jumps there. Hover (desktop) / long-press (phone) → inline SAVE ROLE and FORK AGENT actions — the timeline becomes the primary entry point for both.

Live behavioral mock — this is the contract, interactively

active: T4 · live

Try it: press a diamond → the context jumps and the span highlights. Scroll the context → the active diamond steps with you (sync guard prevents loops). Hover a diamond → title (in the selected mode) + SAVE ROLE / FORK AGENT. The green dot is the live head.

Interaction rules

  • Stops are evenly spaced when they fit; beyond ~14 checkpoints the timeline becomes proportional (position ∝ scrollLine) with collision-nudging so stops never overlap. Tap targets stay ≥44px (phone).
  • Press = jump + select. Selection state = amber-filled diamond + highlight span in the context.
  • Hover/long-press popover: title (current mode), timestamp, and the two actions. Actions call the existing role-save-from-checkpoint and fork flows with the stop's turn — no new backend.
  • Long-press: 350 ms hold, cancels on 10px drift; popover dismisses on outside tap / Esc / scroll.
  • The ◀/▶ pager buttons are removed from the overlay header; keyboard ←/→ still step checkpoints (a11y parity), Esc returns to live.
  • Jumping back while a turn streams: the live head keeps pulsing; any new output auto-scroll is suspended while a past checkpoint is selected and resumes on return-to-live (press the live dot or Esc).

Fresco visual notes

  • Diamond = saved checkpoint (amber when active), pulsing round dot = live head, violet ring on a stop whose turn seeded a role — reuse the checkpoint badge + RoleAvatar language.
  • Highlight span: low-alpha amber wash + 2px left rule (see mock) — readable on the black view without fighting the phosphor text.
  • Phone: timeline width ≥48px, popover becomes a bottom sheet under 480px width.
