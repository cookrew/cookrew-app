Cookrew — an open-source Maestri

A working Electron replica of Maestri's spatial agent workspace: terminals, notes and browser portals on an infinite canvas, orchestrated by a Unix-socket CLI — researched against the real app from inside one of its own terminals, built and verified end-to-end in one session.

Architecture

Three Maestri design principles were kept intact: the CLI is the whole agent API (no SDK — a binary on PATH); connectivity is authorization (an agent can only reach nodes wired to its terminal); and Maestro-gating keeps canvas-restructuring verbs (recruit, dismiss, connect, routines) away from ordinary agents.

Second-pass verification — computer-use sweep

A full regression sweep was run after the initial build: every CLI verb against the live app, then the UI itself driven with real injected input events. macOS blocked System-Events automation (TCC consent needs a human), so a debug pair was added to the app instead — cookrew app-shot (window capture) and cookrew ui click/dblclick/type/key, which injects events through Electron's genuine input pipeline (same hit-testing and handlers as user input).
