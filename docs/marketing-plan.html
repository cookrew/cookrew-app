Cookrew Market Plan, from First Principles

Product + competitor research done fresh today (15 competitors mapped, live sites fetched). Core finding: "infinite canvas for agents" is no longer a differentiator — it's a category. At least five shipping products have it. What nobody else has: cables that mean permission, a symmetric agent-society CLI, and multiplayer. This plan repositions Cookrew around those three, and supersedes the channel/messaging sections of launch-plan v2 (the phase timeline there still stands).

01First principles: what problem is actually being solved?

Strip away features. A developer running multiple AI coding agents has exactly three scarce resources:

1 · Attention

You can watch one terminal at a time. N agents ⇒ N−1 are unwatched.

Whoever best converts "many agents" into "one glanceable surface" wins the workflow. Space is the oldest attention technology humans have — hence canvas. But canvas alone is now table stakes.

2 · Trust

An unwatched agent with unbounded reach is a liability, not leverage.

The unsolved problem is scoped authority: what may this agent see and touch? Every competitor either gives agents ambient access to everything (AgentGrid) or no inter-agent access at all. Cookrew's answer — the wiring is the permission model — is structurally unique.

3 · Coordination

Agents that can't talk to each other push all integration work back onto the human.

Competitors do hierarchy (lead/coordinator dispatch) or raw plumbing (stdout→stdin pipes). Cookrew ships a symmetric peer protocol: any agent can ask, share notes, drive browsers, recruit, schedule — with zero SDK, just a binary on PATH.

⇒ The physics of Cookrew

  • Space = organization (attention)
  • Cables = authorization (trust)
  • CLI = protocol (coordination)
  • Canvas = shared memory (notes/browsers all agents can use)

Everything we say publicly should reduce to these four equations.

03Positioning: what we say — and what we stop saying

One-liner (external, everywhere):
"Cookrew — the multiplayer kitchen for humans & AI agents. Wire your crew together on one open-source canvas: the cables are the permissions, and every agent speaks one CLI."

Lead with (defensible)

  • "The graph is the ACL" — connectivity-as-authorization. Unclaimed by anyone. Security-conscious devs' first question answered by the UI itself.
  • Agent society, not agent hierarchy — symmetric peer CLI: ask / notes / browsers / recruit / routines. Zero SDK.
  • Multiplayer — humans + agents in one live canvas (Cloud). Zero competitors have it.
  • Local-first + open source (MIT) + cross-platform — as a combo with the above; alone each is matched.

Stop leading with (commoditized)

  • "Infinite canvas for agents" — ≥5 shipping products. It's the category name now, not the pitch.
  • Embedded browsers, notes-on-canvas, voice, mobile companion, multi-harness support, persistence — all matched somewhere. Mention as proof of completeness, never as the headline.
  • "Cables between terminals" — Maestri shipped visible cables. Our angle is what cables mean, never that they exist.
Threat handling (rehearsed, never improvised):
  • Claude Code Agent Teams commoditizes orchestration primitives → position Cookrew as the best surface to run Agent Teams on (terminal nodes host it), never as an alternative. "Bring your own orchestrator; Cookrew is where you watch it work."
  • Cate (MIT, 1.9k★, fast releases) → out-position on meaning (permission/protocol/multiplayer), out-execute on release cadence. Per launch-plan v2 discipline: never name competitors in our materials; answer comparison threads with our design choices only.
  • Electron objection (HN-inevitable) → pre-seeded FAQ + first comment: cross-platform and open source are product prerequisites, native perf work cited.

04Beachhead & audience

Beachhead: developers already running 2+ CLI coding agents daily (Claude Code / Codex / OpenCode) who have felt the pain of alt-tabbing between blind terminals. They exist in exactly countable places (r/ClaudeAI, HN, agent-tooling X, Discords) and they churn tools fast — which is an advantage for a new entrant with a real differentiator.

Explicitly not yet: teams (until Cloud), no-code agent builders, consumer canvas users (Flowith's market). Saying no to these keeps the message sharp.

05Channels & launch playbook

The single most important channel finding: in 2025–26 HN data, every "canvas + AI agents" Show HN flopped (NodePad 5 pts, Magic Canvas 4, Canvo 2, AgentCanvas 1; Crystal tried 6 times, best 8 pts) — while outcome-framed, open-source-in-title launches won big (Omnara 310 pts, Emdash 206, Vibe Kanban 195, Superset relaunch 24→96). We sell the outcome (agents that talk, drive browsers, recruit help), never the medium ("spatial canvas"). HN→GitHub conversion runs ~1.4 stars per upvote (arXiv 2511.04453); one flop is not a verdict — relaunch with sharper framing works.

Vocabulary (July 2026)

  • Use: "agent teams" (Anthropic-blessed, searched), "background agents", "agentic engineering" (Karpathy), "vibe engineering" (Willison). Our cross-agent messaging maps exactly onto "agent teams" — ride the term, then differentiate on what Claude Code does NOT do: cross-vendor (Claude+Codex), spatial topology, browser nodes, cables-as-permissions.
  • Avoid: "swarm", "100 agents", "vibe coding" (now connotes irresponsibility), orchestration-as-promise. The winning register is sober + concrete numbers + "real work" + open source.
  • Sherlock hedge: Anthropic keeps absorbing the "manage sessions" layer (Agent View, May 2026, killed pure dashboards). Everything we lead with must be a thing Claude Code itself won't do.

  • Research confirms zero competitors have humans+agents co-editing one live canvas; category deaths (Terragon, Vibe Kanban cloud) say cloud-only is the wrong wedge. Local-first desktop + Cloud multiplayer is the hedge. v2 Phase 3 plan stands; waitlist collects from day 1.
