Observability: the event log & reminder cards

One durable, cross-workspace record of everything that happens on the canvas — surfaced as a tiny reminder card the moment it happens, and queryable forever after.

01 Why this exists

Today a teammate, note, or browser lives only inside its workspace's state. Identity resolution (store.node, connectedTo) reads the active workspace only — so switching workspaces or rebooting the app silently loses sight of everything elsewhere. The user can't tell what happened in a workspace they aren't looking at, and the orchestrator can't find teammates it spawned somewhere else.

The fix and the feature are the same object: a single append-only log of canvas events, written cross-workspace and read three ways. Every projection — the reminder card, the metrics panel, the agent roster — is a view over that one log. Build the log first; the rest fall out of it.

02 Architecture

Every mutating store operation emits through one place. Events are batched to disk and fanned out to live consumers. Nothing writes the log except the choke-point; nothing bypasses it.

emit → append (batched, rolled) → fan out to live stream + query API → three read-only projections

03 Substrate — the event log Forge

An append-only JSONL file at ~/.cookrew/events.jsonl. A single store-level emitter is the only writer: every mutating operation emits through it, so the log can never diverge from state. One event, one line.

05 Projection A — reminder cards Velvet · Fresco

A tiny reminder card pops at the top-level Cookrew view the instant an event arrives, then vanishes (~5s). Small and unobtrusive — a status-tinted dot, one line of title, one mono line of context. Events in a workspace you aren't looking at read “in <workspace>” — that's the whole point.

Live preview — this is the component, not a picture

Burst coalescing: events of the same type+workspace within a ~2 s window collapse into one card with a count — fire the burst button to see 11 recruits become a single card. Prevents the stacked-overlay / stream-starvation pain we hit before.

Rules

  • Auto-dismiss ~5s; newest at the bottom of a short stack; cap the stack (e.g. 4) and drop oldest.
  • Coalesce same type+workspaceId within ~2s → one card, count badge.
  • Status-tinted dot reuses the existing phase palette (idle/working/attention) where relevant.
  • Honor prefers-reduced-motion — fade, no slide.

06 Projection B — metrics & history Velvet · Fresco

A panel over the query API: a scrollable timeline of events plus rolled-up counts (agents spawned, cards created, forks, switches) filterable by workspace, type, and time-range. Phone-friendly. This is the “full-sight” view — the durable record the reminder card only glimpses.

07 Projection C — agent roster Forge query Velvet panel

Not a new store — a query: terminals created/recruited minus dismissed/killed, over the whole log, across all workspaces. This is the durable cross-workspace directory that fixes reboot / lose-teammates: identity resolution falls back to it when the active workspace can't answer. If an agents.json was already built, keep it only as a materialized cache of this same query.
